# MeetLens - アプリケーション構想

## 概要

会議の質を可視化・蓄積するWebアプリケーション。
議事録の自動生成だけでなく、会議そのものの品質をスコアリング・可視化し、組織の会議文化を改善することが目的。

個人開発プロジェクト。将来的に社内発表を見据えている。

## 背景・課題

- 多くの組織では会議をやりっぱなしで振り返らない
- 「なんとなくダメな会議」を定量化する手段がない
- 議事録生成だけならCopilot等で足りる → 差別化は「会議の質の可視化」
- 会議データの蓄積による組織分析
- 測定するだけで行動が変わる（ホーソン効果）。ツールの存在自体が介入になる

## 入力データ

- 会議ツール（Teams, Zoom, Webex等）から出力される **.vtt（WebVTT）ファイル**を共通入力フォーマットとする
- VTTは話者名＋タイムスタンプ付きのテキストデータ
- Teams, Zoom, Webexいずれも.vttでエクスポート可能
- 手動アップロード方式（MVPスコープ）

## MVP ユースケース

### 1. 認証・テナント管理
- Cognitoによるログイン
- テナント分離（将来的に複数チームで利用可能にするため、データ設計時点でテナントIDを組み込む）

### 2. VTTアップロード
- .vttファイルをアップロードする
- アップロード時に以下のメタデータを入力：
  - 会議タイトル
  - 会議タイプ（後述）
  - 開催日時

### 3. 会議タイプの選択
会議タイプごとに評価指標の重み付けを変える。
MVPで対応する会議タイプ（いずれも繰り返し開催されるもの）：

| 会議タイプ | 説明 | 重視する指標 |
|---|---|---|
| 定例 | 週次・月次のプロジェクト会議 | 意思決定数、アクションアイテム、脱線率 |
| デイリースクラム | 短時間の進捗共有 | 時間の短さ、発言の均等さ |
| 1on1 | 上司と部下の対話 | 部下の発言比率（7割以上が理想）、アクションアイテム |
| ブレスト・ふりかえり | 創造・探索型の定期開催 | 新規アイデア数、発言者の多様性 |

### 4. 議事録自動生成
- VTTをパースし、LLM（Bedrock）に投げて議事録を生成
- 日本語の誤認識はLLMに文脈から補正させる

### 5. 会議の質分析・スコア化
VTTから以下の指標を抽出・分析する。会議タイプに応じて評価の重み付けを変える。

#### アウトプット系（会議で何が生まれたか）
- **意思決定数**：3分類で抽出
  - 明確な決定（「〜にする」「〜で行く」＋担当者or期限）
  - 保留・先送り（「次回持ち越し」「要検討」）
  - 未決のまま流れた議題
- **アクションアイテムの具体性**：担当者・期限の有無を判定
- **新規論点・アイデアの検出**：報告会で終わっていないかの判定

#### プロセス系（会議の進め方）
- **発言量の偏り**：話者ごとの発言時間・回数の分布
- **脱線率**：議題と無関係な発言の割合
- **時間あたりの議題消化率**

### 6. 発言内容のファクトチェック（外部情報）
- LLMが発言内容から「事実主張」を抽出
- Web検索APIで外部情報と照合
- 結果を「確認済み / 要確認 / 矛盾あり」でタグ付け
- MVPでは外部情報（Web上で検証可能な情報）のみ対象

### 7. 分析結果のHTML生成・ストック
- 議事録＋分析結果をHTMLとして生成
- 会議ごとにストックし、いつでも閲覧可能にする

### 8. 過去の会議一覧閲覧
- テナント内の会議一覧を表示
- 会議タイプでのフィルタリング
- 各会議のスコアサマリーを一覧で確認

## 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Vue 3 + TypeScript + PrimeVue |
| バックエンド | Python Lambda |
| AI/LLM | Amazon Bedrock |
| 認証 | Amazon Cognito |
| データベース | Amazon DynamoDB |
| ファイルストレージ | Amazon S3 |
| IaC | SAM（サーバーレス部分）/ Terraform（基盤部分） |
| CI/CD | AWS Code系サービス |
